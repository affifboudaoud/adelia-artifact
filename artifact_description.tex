\section{Artifact Identification}
% ============================================================

This artifact supports three contributions presented in the paper:

\begin{itemize}
  \item \textbf{C1:} A structure-preserving backward pass for
    block-tridiagonal-arrowhead (BTA) Cholesky factorization that avoids
    materializing the full lower-triangular factor, reducing memory from
    $\mathcal{O}(n_t \cdot n_s^2)$ to $\mathcal{O}(n_s^2)$.
  \item \textbf{C2:} Extension of C1 to multivariate coregional models
    with $n_v$ response variables, including a two-phase distributed
    algorithm for the fused Cholesky--forward-substitution--backward-substitution--selected-inversion pipeline.
  \item \textbf{C3:} Multi-GPU distributed automatic differentiation using
    MPI-based allreduce operations fused into the JAX computation graph
    via \texttt{mpi4jax}, enabling strong and weak scaling.
\end{itemize}

\section{Experiment Workflow}
% ============================================================

Each experiment directory contains:
\begin{itemize}
  \item \texttt{run.py} --- Python experiment script
  \item \texttt{run\_inner.sh} --- Shell wrapper (environment setup + \texttt{srun})
  \item \texttt{run.sbatch} --- SLURM batch submission script
\end{itemize}

\subsection{Quick Validation ($\sim$2 minutes, 1 node)}

\begin{lstlisting}[language=bash]
cd validation && bash validate_quick.sh
\end{lstlisting}

\noindent
Runs the gst\_small model with both automatic differentiation (AD) and
finite differences (FD), verifying:
\begin{itemize}
  \item Forward objective values match ($< 10^{-6}$ relative error)
  \item Gradient vectors agree ($< 10\%$ max relative error; FD has $O(\epsilon^2)$ truncation error)
\end{itemize}

\subsection{Gradient Correctness Validation ($\sim$30 minutes, 1 node)}

\begin{lstlisting}[language=bash]
cd validation
sbatch validate_gradient_correctness.sbatch
\end{lstlisting}

\noindent
Cross-validates the custom backward pass (AD-BTA) against JAX's native
reverse-mode AD (AD-Loop) on the four models where AD-Loop fits in memory
(GST-S, GST-C2, GST-C3, GST-M).
Both methods are exact (no truncation error), so agreement proves
correctness.
Verifies the paper claim (Section~V): the two exact AD methods agree
to below $10^{-7}$ in the worst case.
Results are written to \texttt{validation/reference\_outputs/gradient\_correctness.csv}.

\subsection{Full Reproduction}

\begin{lstlisting}[language=bash]
# Edit YOUR_ACCOUNT in all .sbatch files
bash run_all_experiments.sh
\end{lstlisting}

\noindent
This submits all experiments to SLURM.
Total estimated wall-clock time: $\sim$28 node-hours.

\section{Validation Criteria}
% ============================================================

\noindent
Numerical correctness is verified by comparing the AD gradient
$\nabla_\theta f_{\mathrm{AD}}$ against a central finite-difference
reference $\nabla_\theta f_{\mathrm{FD}}$ at the initial hyperparameter
values:

\begin{itemize}
  \item \textbf{Forward value:} $|f_{\mathrm{AD}} - f_{\mathrm{FD}}| / |f_{\mathrm{FD}}| < 10^{-6}$
  \item \textbf{Gradient:} $\max_i |\nabla_i f_{\mathrm{AD}} - \nabla_i f_{\mathrm{FD}}| / (|\nabla_i f_{\mathrm{FD}}| + 10^{-10}) < 0.1$
\end{itemize}

\noindent
The gradient tolerance is deliberately loose because central finite
differences have $O(\epsilon^2)$ truncation error with step size
$\epsilon = 10^{-3}$, which is the dominant error source.

Performance metrics (wall-clock time, speedup, energy) are reported as
mean $\pm$ standard deviation over 10 timed gradient evaluations,
after 2 warmup iterations.
